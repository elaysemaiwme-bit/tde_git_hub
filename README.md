# tde_git_hub

# TDE: Git e GitHub - Respostas dos Cursos

Este repositório contém as respostas das questões apresentadas nos cursos de Git e GitHub, conforme solicitado no TDE.

## Seção 1: Introdução

### 1. Que tipo de software é o Git?
**Resposta:** Controle de Versão (Version Control).

### 2. Qual característica NÃO é uma diferença entre Git e GitHub?
**Resposta:** O GitHub contém um terminal especial na web para interagir com seu repositório.

### 3. Qual é o melhor terminal para usar em todos os sistemas operacionais?
**Resposta:** Bash.

##  Seção 2: The Terminal 

### 1. Qual é o comando correto para voltar um nível (pasta anterior)?
**Resposta:** cd ..

### 2. Como eu criaria uma pasta chamada "dir" no terminal?
**Resposta:** `mkdir dir`

### 3. O que o símbolo ~ representa no terminal?
**Resposta:** O caminho padrão/diretório pessoal.

### 4. Qual é o comando correto para mudar de diretório (entrar em pastas)?
**Resposta:** `cd`

### 5. Como os diretórios são removidos no terminal?
**Resposta:** `rm -rf dir`

##  Seção 3: Git basics 

### 1. O que é o HEAD?
**Resposta:** O commit selecionado atualmente.

### 2. Verdadeiro ou Falso: Você pode fazer um commit assim que fizer as alterações?
**Resposta:** Falso..

### 3. O que o comando 'git add .' faz?
**Resposta:** Adiciona todos os arquivos à área de staging.

### 4. Verdadeiro ou Falso: 'git rm -f' remove o arquivo do diretório de trabalho?
**Resposta:** Verdadeiro.

### 5. Qual é o comando usado para criar um novo repositório?
**Resposta:** git init

##  Seção 4: Git branches

### 1. Como você adiciona e muda para uma nova branch no Git?
**Resposta:** git checkout -b <nome>

### 2. Verdadeiro ou Falso: Mesclar branches une todos os commits em uma única linha do tempo?
**Resposta:** Falso. 

### 3. O que é uma branch?
**Resposta:** Uma cadeia de commits que não interfere em outras branches.

### 4. Verdadeiro ou Falso: Você pode ter vários ponteiros HEAD quando tem mais de uma branch?
**Resposta:** Falso.

### 5. Escolha uma alternativa que NÃO é um benefício de usar branches:
**Resposta:** Aumentar a complexidade do seu projeto.

## Seção 5: GitHub 

### 1. Verdadeiro ou Falso: Forking é a mesma coisa que Pulling?
**Resposta:** Falso. 

### 2. O que é Forking?
**Resposta:** Clonar um repositório do GitHub para os seus próprios repositórios do GitHub.

### 3. Qual é a principal característica do GitHub?
**Resposta:** Permitir que colabores com outras pessoas.

### 4. O que é o GitHub?
**Resposta:** Uma plataforma de hospedagem Git.

### 5. Verdadeiro ou Falso: Repositórios do GitHub podem ser privados?
**Resposta:** Verdadeiro.

## Seção 6: Using Git Remotely

### 1. Qual é o comando usado para adicionar um remote origin?
**Resposta:** git remote add origin <url>

### 2. Verdadeiro ou Falso: 'git remote -v' mostra todos os seus remote origins?
**Resposta:** Verdadeiro. 

### 3. O que é um remote origin?
**Resposta:** O link para o seu repositório remoto.

### 4. Quantas branches você pode dar push por vez para um repositório remoto?
**Resposta:** 1.

### 5. Como você interage com um repositório do GitHub?
**Resposta:** Através de um Ciclo de Push/Pull.

## Seção 7: Git GUI w/ SourceTree

### 1. Verdadeiro ou Falso: Você precisa de uma conta para usar o SourceTree?
**Resposta:** Verdadeiro.

### 2. Que tipo de software é o SourceTree?
**Resposta:** GUI.

### 3. Escolha uma opção que NÃO é um benefício de usar o SourceTree:
**Resposta:** Melhor performance.

### 4. Verdadeiro ou Falso: É muito difícil adicionar remote origins através do SourceTree?
**Resposta:** Verdadeiro.

### 5. O que o 'Git-flow' faz?
**Resposta:** Configura um modelo (template) para você usar em seu repositório.
